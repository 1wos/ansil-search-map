

# 안심찾기 — 구현 계획

## Phase A-1: 메인 페이지 레이아웃 (히어로 + 통계 카드)

### 디자인 시스템 설정
- `index.css`에 커스텀 색상 변수 적용 (Soft Coral `#E8756A`, Deep Navy `#1E3A5F`, Warm Amber `#F5A623` 등)
- Pretendard / Noto Sans KR 웹폰트 연결
- 카드, 버튼, 배지 등 공통 스타일 토큰 정의

### 메인 페이지 (`/`) 구성
1. **네비게이션 바** — 로고("안심찾기"), 메뉴(지도, 제도 목록, AI 상담), 모바일 햄버거 메뉴
2. **히어로 섹션** — 따뜻한 톤의 헤드라인("내 지역 안심 지원제도, 한눈에 찾아보세요"), 부제목, AI 검색 입력창(CTA), 배경에 은은한 일러스트/그라데이션
3. **핵심 통계 카드 (3~4개)** — 전국 지원제도 수, 참여 지자체 수, 카테고리 수 등을 mock 데이터로 표시. 아이콘 + 숫자 + 라벨 구성
4. **지도 영역 플레이스홀더** — "여기에 전국 지도가 들어갑니다" 안내 (A-2에서 SVG 지도 구현)
5. **푸터** — 서비스 소개, 문의, 저작권 표시

### 모바일 우선 반응형
- 모바일: 세로 1컬럼 레이아웃, 통계 카드 2×2 그리드
- 데스크톱(`md` 이상): 히어로 좌우 배치, 통계 카드 가로 4열

### Mock 데이터
- 통계 수치(예: 전국 1,247개 제도, 17개 광역시도, 8개 카테고리)를 상수로 정의

## Phase A-2: SVG 한국 지도 컴포넌트
- 17개 광역시·도 SVG path 데이터로 클릭 가능한 인터랙티브 지도 구현
- 호버 시 지역명 툴팁 + 색상 하이라이트
- 클릭 시 사이드 패널에 해당 지역 제도 요약 표시 (mock 데이터)
- "이 지역 전체 보기" 버튼 → `/region/:cityName` 라우팅

## Phase A-3: 지역 상세 페이지
- `/region/:cityName` 라우트 생성
- 지역 헤더(지역명 + 제도 건수 + 카테고리 필터 탭)
- 제도 카드 그리드: 카드마다 제도명, 카테고리 배지, 대상, 지원내용 요약, [신청하기] 버튼
- Mock 제도 데이터 10~15건

## Phase A-4: 챗봇 UI
- `/chat` 라우트 + 메인 페이지 하단 미니 검색창
- 대화형 인터페이스: 사용자 메시지(우측), AI 응답(좌측) 말풍선
- AI 응답에 제도 카드 임베드 + 신청 링크
- Mock 응답으로 UI 완성

## Phase A-5: 제도 목록 페이지
- `/programs` 라우트
- 필터바: 지역, 카테고리, 대상 조건 드롭다운
- 검색 입력창 + 제도 카드 리스트
- Mock 데이터 20~30건
